## Database Setup

### SQLite to Production Migration

#### Option 1: Keep SQLite (Small to Medium Deployments)

```bash
# Initialize database
cd backend
npm run seed

# Backup database
cp database.sqlite database.sqlite.backup.$(date +%Y%m%d)

# Set proper permissions
chmod 644 database.sqlite
```

**Configuration (.env):**
```
DB_PATH=./database.sqlite
```

**Backup Strategy:**
- Daily backups: `0 2 * * * cp /path/to/database.sqlite /backups/db_$(date +\%Y\%m\%d).sqlite`
- Store in separate location
- Test restore weekly

---

#### Option 2: Migrate to PostgreSQL (Recommended for Large Deployments)

**Advantages:**
- Better concurrency
- More robust
- Built-in replication
- Better query optimization

**Steps:**
1. Install PostgreSQL
2. Create database and user:
```sql
CREATE DATABASE leave_management;
CREATE USER leave_app WITH PASSWORD 'secure_password';
ALTER ROLE leave_app WITH CREATEDB;
GRANT ALL PRIVILEGES ON DATABASE leave_management TO leave_app;
```

3. Update database.js to use PostgreSQL (pg driver)
4. Migrate schema to PostgreSQL
5. Update .env:
```
DB_HOST=localhost
DB_PORT=5432
DB_NAME=leave_management
DB_USER=leave_app
DB_PASSWORD=secure_password
DB_SSL=true
```

## Backend Deployment

### Option 1: Traditional Server (Linux/Ubuntu)

#### Prerequisites
- Node.js 16+ installed
- npm or yarn
- Port 5000 available (or configured)

#### Step 1: Deploy Code
```bash
# Clone/copy repository
git clone <repo-url> /opt/leave-app
cd /opt/leave-app/backend

# Install dependencies
npm install --production

# Create required directories
mkdir -p logs uploads
```

#### Step 2: Configure Environment
```bash
# Create .env file
cat > .env << EOF
NODE_ENV=production
PORT=5000
JWT_SECRET=$(openssl rand -base64 32)
DB_PATH=/data/database.sqlite
LOG_LEVEL=info
SMTP_HOST=smtp.gmail.com
SMTP_PORT=587
SMTP_USER=notifications@company.com
SMTP_PASSWORD=app_password
SMTP_FROM_NAME=Leave Management System
TWILIO_ACCOUNT_SID=your_sid
TWILIO_AUTH_TOKEN=your_token
TWILIO_PHONE_NUMBER=+1234567890
EOF

chmod 600 .env
```

#### Step 3: Setup Process Manager (PM2)
```bash
# Install PM2
npm install -g pm2

# Create ecosystem.config.js
cat > ecosystem.config.js << 'EOF'
module.exports = {
  apps: [{
    name: 'leave-app-backend',
    script: './src/server.js',
    instances: 'max',
    exec_mode: 'cluster',
    env: {
      NODE_ENV: 'production'
    },
    error_file: './logs/error.log',
    out_file: './logs/out.log',
    log_date_format: 'YYYY-MM-DD HH:mm:ss Z'
  }]
};
EOF

# Start application
pm2 start ecosystem.config.js

# Setup autostart
pm2 startup
pm2 save
```

#### Step 4: Setup Reverse Proxy (Nginx)
```bash
# Install Nginx
sudo apt-get install nginx

# Create config
sudo cat > /etc/nginx/sites-available/leave-app << 'EOF'
upstream leave_backend {
    server localhost:5000;
    keepalive 64;
}

server {
    listen 80;
    server_name api.leaveapp.com;

    # Redirect HTTP to HTTPS
    return 301 https://$server_name$request_uri;
}

server {
    listen 443 ssl http2;
    server_name api.leaveapp.com;

    ssl_certificate /etc/letsencrypt/live/api.leaveapp.com/fullchain.pem;
    ssl_certificate_key /etc/letsencrypt/live/api.leaveapp.com/privkey.pem;

    # Security headers
    add_header Strict-Transport-Security "max-age=31536000" always;
    add_header X-Frame-Options "SAMEORIGIN";
    add_header X-Content-Type-Options "nosniff";
    add_header X-XSS-Protection "1; mode=block";

    # Gzip compression
    gzip on;
    gzip_types text/plain text/css application/json application/javascript;

    location / {
        proxy_pass http://leave_backend;
        proxy_http_version 1.1;
        proxy_set_header Connection "";
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;

        # Timeouts
        proxy_connect_timeout 60s;
        proxy_send_timeout 60s;
        proxy_read_timeout 60s;
    }

    # Static file uploads
    location /uploads {
        alias /opt/leave-app/backend/uploads;
        expires 30d;
        add_header Cache-Control "public, immutable";
    }
}
EOF

# Enable site
sudo ln -s /etc/nginx/sites-available/leave-app /etc/nginx/sites-enabled/
sudo nginx -t
sudo systemctl restart nginx
```

### Option 2: Docker Deployment

#### Create Dockerfile
```dockerfile
# backend/Dockerfile
FROM node:18-alpine

WORKDIR /app

COPY package*.json ./
RUN npm install --production

COPY src/ ./src/
COPY .env ./

# Create required directories
RUN mkdir -p uploads logs

EXPOSE 5000

# Health check
HEALTHCHECK --interval=30s --timeout=10s --start-period=40s --retries=3 \
    CMD node -e "require('http').get('http://localhost:5000/health', (r) => {if (r.statusCode !== 200) throw new Error(r.statusCode)})"

CMD ["node", "src/server.js"]
```

#### Create docker-compose.yml
```yaml
version: '3.8'

services:
  backend:
    build: ./backend
    container_name: leave-app-backend
    ports:
      - "5000:5000"
    environment:
      - NODE_ENV=production
      - PORT=5000
    volumes:
      - ./backend/database.sqlite:/app/database.sqlite
      - ./backend/uploads:/app/uploads
      - ./backend/logs:/app/logs
    restart: always
    networks:
      - leave-app

  frontend:
    build: ./frontend
    container_name: leave-app-frontend
    ports:
      - "3000:3000"
    environment:
      - REACT_APP_API_URL=https://api.leaveapp.com
    depends_on:
      - backend
    restart: always
    networks:
      - leave-app

  nginx:
    image: nginx:alpine
    container_name: leave-app-nginx
    ports:
      - "80:80"
      - "443:443"
    volumes:
      - ./nginx.conf:/etc/nginx/nginx.conf:ro
      - /etc/letsencrypt:/etc/letsencrypt:ro
    depends_on:
      - backend
      - frontend
    restart: always
    networks:
      - leave-app

networks:
  leave-app:
    driver: bridge
```

#### Deploy
```bash
# Build and start containers
docker-compose up -d

# View logs
docker-compose logs -f backend

# Stop services
docker-compose down
```

## Frontend Deployment

### Option 1: Traditional Hosting

#### Build for Production
```bash
cd frontend
npm run build

# Creates optimized production build in 'build' folder
```

#### Deploy to Nginx
```bash
# Copy build to web root
sudo cp -r frontend/build/* /var/www/leave-app/

# Configure nginx to serve
sudo cat > /etc/nginx/sites-available/leave-app-web << 'EOF'
server {
    listen 443 ssl http2;
    server_name app.leaveapp.com;

    ssl_certificate /etc/letsencrypt/live/app.leaveapp.com/fullchain.pem;
    ssl_certificate_key /etc/letsencrypt/live/app.leaveapp.com/privkey.pem;

    root /var/www/leave-app;
    index index.html;

    location / {
        # SPA routing - serve index.html for all routes
        try_files $uri $uri/ /index.html;
    }

    # Cache static assets
    location ~* \.(js|css|png|jpg|jpeg|gif|ico|svg|woff|woff2|ttf|eot)$ {
        expires 1y;
        add_header Cache-Control "public, immutable";
    }

    # API proxy
    location /api {
        proxy_pass https://api.leaveapp.com;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }
}
EOF
```

### Database Maintenance

#### Regular Backups
```bash
# Daily backup script
#!/bin/bash
BACKUP_DIR="/backups/leave-app"
DATE=$(date +%Y%m%d_%H%M%S)
sqlite3 /opt/leave-app/backend/database.sqlite ".backup $BACKUP_DIR/db_$DATE.sqlite"

# Compress older backups
find $BACKUP_DIR -name "db_*.sqlite" -mtime +7 -exec gzip {} \;

# Remove backups older than 30 days
find $BACKUP_DIR -name "db_*.sqlite.gz" -mtime +30 -delete

# Add to crontab
0 2 * * * /path/to/backup.sh
```

## Backup & Recovery

### Backup Strategy

#### 3-2-1 Backup Rule
- **3 copies** of data
- **2 different storage media** types
- **1 offsite** backup

#### Implementation
```bash
# Daily local backup
0 2 * * * sqlite3 /opt/leave-app/backend/database.sqlite ".backup /backups/leave-app/db_daily.sqlite"

# Weekly offsite backup (S3)
0 3 * * 0 aws s3 cp /backups/leave-app/db_daily.sqlite s3://leave-app-backups/$(date +%Y%m%d).sqlite

# Monthly archive
0 4 1 * * tar czf /archive/leave-app-$(date +%Y%m).tar.gz /opt/leave-app/
```

---

### Recovery Procedures

#### Restore from Backup
```bash
# Stop application
pm2 stop all

# Restore database
cp /backups/leave-app/db_daily.sqlite /opt/leave-app/backend/database.sqlite

# Restart application
pm2 start all

# Verify
curl http://localhost:5000/health
```

#### Full System Recovery
```bash
# 1. Restore server configuration
# 2. Install dependencies
cd /opt/leave-app/backend
npm install --production

# 3. Restore database
cp /backups/leave-app/db_latest.sqlite /opt/leave-app/backend/database.sqlite

# 4. Restore uploads
rsync -av /backups/leave-app/uploads /opt/leave-app/backend/

# 5. Start services
pm2 start ecosystem.config.js
```

## Troubleshooting

### Common Issues

#### Issue: High Memory Usage
**Diagnosis:**
```bash
ps aux | grep node
# Check memory column

pm2 monit
```

**Solutions:**
1. Increase Node memory: `node --max-old-space-size=4096`
2. Check for memory leaks
3. Enable garbage collection monitoring
4. Scale horizontally with load balancer

---

#### Issue: Slow Database Queries
**Diagnosis:**
```bash
# Enable query logging
# Monitor slow queries

# Check indexes
sqlite3 database.sqlite ".indices"
```

**Solutions:**
1. Add missing indexes
2. Optimize queries
3. Increase database cache
4. Consider migration to PostgreSQL

---

#### Issue: SSL Certificate Error
**Solution:**
```bash
# Check certificate validity
openssl x509 -in /etc/letsencrypt/live/api.leaveapp.com/fullchain.pem -text -noout

# Reload Nginx
```

## Rollback Procedure

### Quick Rollback
```bash
# If current version has critical issues:

# 1. Stop application
pm2 stop leave-app-backend

# 2. Restore previous code
git checkout HEAD~1

# 3. Install dependencies
npm install --production

# 4. Restart
pm2 start leave-app-backend

# 5. Monitor
pm2 logs leave-app-backend
```

### Database Rollback
```bash
# If migrations caused issues:

# 1. Stop application
pm2 stop all

# 2. Restore previous database backup
cp /backups/leave-app/db_previous.sqlite /opt/leave-app/backend/database.sqlite

# 3. Restore previous code
git checkout HEAD~1

# 4. Restart
pm2 start all
```
